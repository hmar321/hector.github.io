18-A continuación realizamos la implementación de la interfaz. Es importante recalcar que cada registro de la base de datos se guarda en un objeto java org.bson.Document y este objeto tiene un atributo org.bson.types.ObjectId que nosotros guardaremos en un String id pero habrá que realizar las operaciones de conversión adecuadas en cada caso.

package atlas.dao;

import org.bson.Document;

import org.bson.types.ObjectId;

import com.mongodb.MongoException;

import com.mongodb.client.MongoClient;

import com.mongodb.client.MongoCollection;

import com.mongodb.client.MongoDatabase;

import atlas.model.Vehiculo;

import atlas.util.DatabaseConnection;

public class VehiculoDAOImplementation implements VehiculoDAO {

    @Override

    public void insertVehiculo(Vehiculo vehiculo, MongoClient mongoClient) throws MongoException {

        mongoClient = DatabaseConnection.getMongoClient();

        MongoDatabase database = mongoClient.getDatabase("concesionario");

        MongoCollection<Document> collection = database.getCollection("vehiculos");

        Document document = new Document("matricula", vehiculo.getMatricula())

                .append("modelo", vehiculo.getModelo())

                .append("marca", vehiculo.getMarca());

        collection.insertOne(document);

        ObjectId id = document.getObjectId("_id");

        vehiculo.setId(id.toString());

        mongoClient.close();

    }

    @Override

    public void updateVehiculo(Vehiculo vehiculo, MongoClient mongoClient) throws MongoException {

        mongoClient = DatabaseConnection.getMongoClient();

        MongoDatabase database = mongoClient.getDatabase("concesionario");

        MongoCollection<Document> collection = database.getCollection("vehiculos");

        Document filter = new Document("matricula", vehiculo.getMatricula());

        Document update = new Document("$set",

                new Document("id", vehiculo.getId())

                        .append("matricula", vehiculo.getMatricula())

                        .append("modelo", vehiculo.getModelo())

                        .append("marca", vehiculo.getMarca()));

        collection.updateOne(filter, update);

        mongoClient.close();

    }

    @Override

    public void deleteVehiculoById(String id, MongoClient mongoClient) throws MongoException {

        mongoClient = DatabaseConnection.getMongoClient();

        MongoDatabase database = mongoClient.getDatabase("concesionario");

        MongoCollection<Document> collection = database.getCollection("vehiculos");

        ObjectId objectId = new ObjectId(id);

        Document filter = new Document("_id", objectId);

        collection.deleteOne(filter);

        mongoClient.close();

    }

    @Override

    public Vehiculo findVehiculoById(String id, MongoClient mongoClient) throws MongoException {

        mongoClient = DatabaseConnection.getMongoClient();

        MongoDatabase database = mongoClient.getDatabase("concesionario");

        MongoCollection<Document> collection = database.getCollection("vehiculos");

        ObjectId objectId = new ObjectId(id);

        Document filter = new Document("_id", objectId);

        Document result = collection.find(filter).first();

        if (result == null) {

            return null;

        }

        String vehId = result.getObjectId("_id").toString();

        String matricula = result.getString("matricula");

        String modelo = result.getString("modelo");

        String marca = result.getString("marca");

        Vehiculo recuperado = new Vehiculo(matricula, modelo, marca);

        recuperado.setId(vehId);

        mongoClient.close();

        return recuperado;

    }

}

Definiciones proporcionadas por Chat GPT:

  • MongoClient: Representa una conexión abierta a un servidor MongoDB y se utiliza para realizar operaciones de base de datos, como leer, escribir y eliminar documentos en una o varias colecciones de una base de datos MongoDB.

  • MongoDatabase: Representa una base de datos en MongoDB. Una instancia de MongoClient puede contener varias bases de datos y se utiliza para obtener una instancia de MongoDatabase que se puede utilizar para realizar operaciones en esa base de datos específica.

  • MongoCollection: Representa una colección de documentos en una base de datos MongoDB. Una instancia de MongoDatabase puede contener varias colecciones y se utiliza para obtener una instancia de MongoCollection que se puede utilizar para realizar operaciones en esa colección específica.

  • Document: Representa un documento en una colección de MongoDB. Un documento es la unidad básica de almacenamiento de datos en MongoDB y se compone de campos y valores.

  • ObjectId: Es un tipo de datos utilizado en MongoDB para identificar de forma única un documento en una colección. Se utiliza para generar identificadores únicos para cada documento y se puede utilizar para buscar, actualizar o eliminar documentos específicos en una colección.
